- **Clash Verge Rev detection — Unix socket, not TCP (VERIFIED 2026-06-08, bug-052):** current Clash Verge Rev sets `external-controller: ''` (TCP API OFF) and exposes the mihomo controller ONLY via `external-controller-unix` (e.g. `/tmp/verge/verge-mihomo.sock`). IMPORTANT framing: when Verge runs in **TUN mode** (`/configs` → `tun.enable=true`), the macOS **system proxy is legitimately OFF** and it is **not** a `scutil --nc` VPN — so "System Proxy disabled / no VPN" is CORRECT, not a bug; the proxy client + nodes are where Verge shows up. - [2026-06-07] **TUI byte slicing crash:** `shorten_path()` and `truncate_id()` used `&s[byte_index..]` which panicked on Chinese characters (e.g. '包' = 3 bytes). Fixed to use `chars().skip().collect()`.
- [2026-06-07] **TUI tab index crash:** After `refresh()` updates agents list, `active_tab` could exceed new tab count. Added clamping in both `refresh()` and `draw()`.
- [2026-06-07] **Forgot TUI parity:** Added Proxy/VPN tab to GUI but forgot TUI. Always add features to both interfaces.
- [2026-06-08] **Blank GUI from `cargo run -- gui`:** The fix is NOT `npm run build` then re-run (I claimed that prematurely after an invalid HTTP-only "verification" — the user re-tested and it was still blank). REAL fix: run the GUI via the Tauri CLI — `tauri dev` (`npm run gui`) is verified to render; `cargo run -- gui` serves a stale/mismatched embedded frontend → blank. Lesson: NEVER claim a webview fix verified by loading `build/` over plain HTTP — no Tauri runtime there, so the empty-data shell mounts and hides the real failure. Verify inside the real webview (tauri dev + app.html beacon). (was bug-036)

## Decision Log

- **2026-06-07: Proxy detection approach** — Used `networksetup` commands (not `sysproxy` crate) to avoid adding a dependency. Parse text output for HTTP/HTTPS/SOCKS/PAC settings. Query Clash/Surge REST APIs for node info.
- **2026-06-07: TUI tab structure** — Dashboard + per-installed-agent tabs + Proxy/VPN + Keep-Alive. Dynamic tab count based on installed agents. Key bindings: 1=Dash, 2-9=agents, 0=KA.
- **2026-06-07: GUI tab structure** — Same as TUI: system+network at top, then agent tabs with usage (5h/1w) and sessions, Proxy/VPN tab, Keep-Alive at bottom.
- **2026-06-08: OpenCode integration** — Added as 4th agent at full parity. Read its single Drizzle SQLite DB directly via rusqlite (matches Codex/Cursor pattern; no new deps). Modeled token usage by SUMMING per-call message totals (OpenCode reports per-call, unlike Codex's cumulative). No rate-limit windows (BYO-provider, no unified API) and no GUI split (CLI/TUI-only). Relied on the data-driven GUI/TUI to pick up the new agent automatically rather than touching UI code.
